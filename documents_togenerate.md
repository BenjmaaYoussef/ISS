## Individual documents to submit

# UI navigation diagram

Each interface is represented with rectangle containing the ID and the title.
All those rectangles are linked (with a line) to represent the navigation ability from an interface to another.

# Data model

Use any software to draw the class diagram of the data model (local storage schema in our app).

# Features grouping

Submit a word document stating what are the entities involved in each feature.

You should put the ID of the feature (e.g. F1, F2, ...) as defined in previous documents and state what are the entities involved as defined in the data model.

# Project features & User roles

Submit one document containing the list of all the user roles in your projects (e.g. visitor, admin, ...)

That document should contains the list of all the features of your projects and stating what are the user roles involved in each feature.

Each feature should have an ID (e.g. F1, F2, F3, ...) to be used in later documents.

# Project Scenarios

Submit one document containing the different scenarios (use cases) in your project.

Each scenarios has an ID (e.g. S1, S2, S3, ...)

Each scenario contains a list of steps, with each an ID (e.g. for scenario S1 we have the steps: S1.1, S1.2, S1.3, ...).

For each step, you should associate the involved user roles and features (specify the feature ID as defined in the previous document; see it description).
